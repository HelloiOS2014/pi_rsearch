Core principle: "在 LLM 上下文中，保留是昂贵的，取回是廉价的——与传统计算机内存模型完全相反。"

Use a `CodeComparison` to contrast traditional vs LLM memory models:

```mdx
export const codeTraditionalMemory = `// 传统计算：保留便宜，取回昂贵
page_in_ram = FREE          // 页面在内存中：零成本
page_fault  = EXPENSIVE     // 缺页中断：磁盘 I/O，毫秒级延迟

// 最优策略：尽量多保留，减少缺页`;

export const codeLLMMemory = `// LLM 上下文：保留昂贵，取回便宜
token_in_context = EXPENSIVE  // 每个 token 消耗注意力预算
tool_call_fetch  = CHEAP      // 一次工具调用，毫秒级

// 最优策略：精准保留高信号 token，其余按需取回`;

<CodeComparison client:visible
  left={{ code: codeTraditionalMemory, label: "传统内存模型", language: "typescript" }}
  right={{ code: codeLLMMemory, label: "LLM 上下文模型", language: "typescript" }}
/>
```

Then: unified table showing cost inversion across harness subsystems (tool schemas, skills, history, file content, subtask context).

Conclude with heuristic: "在 Harness 的每一层，默认选择'按需取回'而非'常驻保留'，除非有证据表明取回延迟不可接受。"
