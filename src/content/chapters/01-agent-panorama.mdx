本章目标：从零理解 Agent 是什么，看懂 Pi 的三层架构，最后用 30 行代码写出一个最小 Agent。

## 1.1 什么是 Agent

### 一句话定义

**Agent = LLM（决策者）+ Tools（执行者）+ Loop（协调者）**

这不是三个独立概念的拼凑，而是一个完整的运行时系统。LLM 负责思考和决策，Tools 负责与外部世界交互，Loop 把两者粘合成一个持续运行的闭环。

### ReAct 模式

Agent 的核心运行模式是 **ReAct**（Reasoning + Acting）：

1. **Reasoning**：LLM 分析当前状态，决定下一步做什么
2. **Acting**：调用工具执行动作
3. **Observing**：观察工具返回的结果
4. **循环**：基于观察结果继续推理，直到任务完成

```
用户提问 → LLM 思考 → 调用工具 → 观察结果 → LLM 再思考 → 调用工具 → … → 最终回复
```

### 与 ChatBot 的本质区别

Agent 不是"更聪明的 ChatBot"，是根本不同的范式：

| | ChatBot | Agent |
|---|---|---|
| 交互模式 | 一问一答 | 自主执行多步任务 |
| 工具使用 | 无 | 自主决定何时调用什么工具 |
| 状态管理 | 无状态对话 | 维护执行上下文 |
| 停止条件 | 回复即结束 | 任务完成才结束 |
| 核心能力 | 生成文本 | **自主决策 + 执行动作** |

Agent 的核心能力是 **自主决策**：它自己判断何时使用什么工具，观察结果后决定下一步行动。这不是预编程的 workflow，而是 LLM 在运行时动态规划的执行路径。

## 1.2 Pi 架构鸟瞰

Pi 是一个开源的 coding agent，用 TypeScript 实现，代码量约 1,900 行核心代码。它的架构分为三层，每层职责清晰：

```
┌─────────────────────────────────────────────┐
│  pi-coding-agent                            │
│  7 个内置工具 · 扩展系统 · 会话管理         │

│  pi-agent-core                              │
```

### 第三层：pi-coding-agent — 完整的 Coding Agent

这一层在 Agent 运行时之上，构建了一个可以读写代码、执行命令的完整 coding agent：

- **7 个内置工具**：`read`、`bash`、`edit`、`write`、`grep`、`find`、`ls`
- **扩展系统**：通过配置文件添加自定义工具和 system prompt
- **会话管理**：持久化对话历史，支持恢复中断的会话
- **上下文压缩**：当对话超过 context window 时自动压缩
- **3 种运行模式**：
  - `Interactive`：终端交互模式（类似 Claude Code）
  - `Print`：单次执行，输出结果后退出
  - `RPC`：作为后端服务运行

## 1.4 动手：30 行最小 Agent

现在我们从零开始，分三步构建一个最小 Agent。通过对比，你会清晰地看到 Agent 和普通 LLM 调用的本质区别。

### Step 1: 裸调 LLM

最简单的 LLM 调用 — 发送一个问题，得到一个回复：

```typescript
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic();
const response = await client.messages.create({
  model: "claude-sonnet-4-5-20250514",
  max_tokens: 1024,
  messages: [{ role: "user", content: "什么是 Agent？" }],
});
console.log(response.content[0].text);
```

这就是一个 ChatBot — 一问一答，没有工具，没有循环。

### Step 2: 加入工具定义

给 LLM 提供工具定义，让它有能力"请求"调用工具：

```typescript
const tools = [{
  name: "calculator",
  description: "计算数学表达式",
  input_schema: {
    type: "object" as const,
    properties: {
      expression: { type: "string", description: "数学表达式" }
    },
    required: ["expression"]
  }
}];

const response = await client.messages.create({
  model: "claude-sonnet-4-5-20250514",
  max_tokens: 1024,
  tools,
  messages: [{ role: "user", content: "计算 237 * 891 + 42" }],
});

// 关键！看 stop_reason
console.log(response.stop_reason); // "tool_use" — LLM 决定调用工具！
```

注意 `stop_reason` 变成了 `"tool_use"` — LLM 没有直接回答，而是请求调用 calculator 工具。但此时工具并没有真正执行，因为缺少最关键的部分：**循环**。

### Step 3: 完整 Agent Loop

加上循环，就是一个完整的 Agent：

```typescript
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic();

// 工具定义 + 执行逻辑
const tools = [/* ... 同上 ... */];
function executeMyTool(name: string, input: any): string {
  if (name === "calculator") {
    return String(eval(input.expression)); // 仅做演示！
  }
  return "Unknown tool";
}

// 🔥 Agent 的核心：一个 while 循环
const messages: any[] = [{ role: "user", content: "计算 237 * 891 + 42" }];

while (true) {
  const response = await client.messages.create({
    model: "claude-sonnet-4-5-20250514",
    max_tokens: 1024,
    tools,
    messages,
  });

  // 追加 assistant 回复到上下文
  messages.push({ role: "assistant", content: response.content });

  // 没有工具调用 → 任务完成，退出循环
  if (response.stop_reason !== "tool_use") break;

  // 执行每个工具调用，将结果追加到上下文
  const toolResults = [];
  for (const block of response.content) {
    if (block.type === "tool_use") {
      const result = executeMyTool(block.name, block.input);
      toolResults.push({
        type: "tool_result",
        tool_use_id: block.id,
        content: String(result),
      });
    }
  }
  messages.push({ role: "user", content: toolResults });
}
```

## 1.5 本章小结

三个核心认知：

1. **Agent 的本质是一个 while 循环** — 让 LLM 能够多轮调用工具、观察结果、持续推理
2. **Pi 的三层架构**：`pi-ai`（统一 LLM 接口）→ `pi-agent-core`（Agent 运行时）→ `pi-coding-agent`（完整 Coding Agent）
3. **Pi 用约 1,900 行核心代码证明了这个循环可以做到和 Claude Code 竞争** — 架构简洁不等于功能简单
